# Как редактировать сайт без кода

## Базовый сценарий

1. Откройте `admin.html`.
2. Перейдите в нужный раздел редактора.
3. Меняйте тексты, названия, описания, пути к изображениям, статусы и переключатели.
4. Все изменения сразу сохраняются в браузере.
5. Нажмите `Экспорт JSON`.
6. Скачанный файл замените на `content/content.json`.

## Что можно редактировать

### Главный экран

- Заголовок по строкам.
- Подзаголовок.
- Роли.
- Дисциплины.
- Технические подписи.

### Кейсы

- Название, slug, клиент, год, роль, категория.
- Summary, challenge, approach, outcome, notes.
- Featured и visible.
- Порядок секций.
- Добавление новых секций.
- Удаление секций.
- Пути к изображениям.
- Списки услуг.

### Разделы

- Включение и выключение блоков сайта.
- Списки в «Брендинге» и «Арт-дирекшне».

### About / Contacts

- Позиционирование.
- Компетенции.
- Направления.
- Email, Telegram, услуги.

### UI / System

- Название сайта.
- System ID.
- Статус.
- Эффекты noise / scanlines.

## Добавление нового кейса

1. Откройте вкладку `Кейсы`.
2. Нажмите `Добавить кейс`.
3. Заполните основные поля.
4. Добавьте секции: текст, галерею, split-блок или показатели.
5. При необходимости включите `featured`.
6. Экспортируйте JSON.

## Скрытие кейса

Снимите чекбокс `Показывать кейс`. Кейс останется в JSON, но исчезнет с сайта.

## Как менять порядок

- Для массивов есть кнопки `↑` и `↓`.
- Так можно менять порядок кейсов, услуг, секций и карточек.

## Как использовать видео

Для секции типа `video` вставляйте embed URL, например для Vimeo или YouTube embed.

## Лучший рабочий процесс

- Сначала редактируйте всё в `admin.html`.
- Потом экспортируйте JSON.
- Проверяйте сайт в `index.html`.
- После проверки переносите новый `content.json` на хостинг.
