# Как редактировать сайт без кода

## Базовый сценарий

1. Откройте `admin.html` (лучше всего — по адресу задеплоенного сайта, тогда работают загрузка
   картинок и кнопка «Опубликовать»).
2. Перейдите в нужный раздел редактора (вкладки слева / сверху на мобильном).
3. Меняйте тексты, названия, описания, статусы и переключатели. Индикатор вверху показывает
   состояние: «Сохранение…» → «Сохранено локально».
4. Все изменения сразу сохраняются в браузере (localStorage) и применяются на сайте.
5. Нажмите «Опубликовать на сайт», чтобы залить контент на хостинг (спросит пароль один раз
   за сессию), **или** «Экспорт JSON» и вручную заменить `content/content.json`.

## Новое и удобное

- **Плавный ввод.** Поля больше не «прыгают» — можно спокойно печатать длинные тексты.
- **Выпадающие списки** для категории кейса, статуса, типа секции и привязки проекта к кейсу —
  меньше опечаток.
- **Превью картинок.** Рядом с каждым полем изображения — миниатюра. Битый путь показывается
  штриховкой-заглушкой.
- **Загрузка файлов.** Кнопка «Загрузить» у каждого поля картинки. Крупные фото автоматически
  сжимаются, файл коммитится в репозиторий (`images/uploads/…`), путь подставляется сам.
  ⚠️ Работает только на задеплоенном сайте (нужен серверный доступ), не при открытии файла с диска.
  Каждая загрузка — отдельный коммит и редеплой.
- **Живое превью.** Кнопка «Показать превью» открывает панель с сайтом рядом с редактором; она
  сама обновляется через ~полсекунды после правки. Есть «Обновить» и «В новой вкладке».
- **Поиск по кейсам** — поле над списком кейсов.
- **Мобильный режим** — редактор адаптирован под телефон.

## Что можно редактировать

### Главный экран
Надстрочник, подзаголовок, метки, строки заголовка, дисциплины.

### Кейсы
Название, slug, клиент, год, роль, категория (список), статус (список), обложка (превью+загрузка),
summary / challenge / approach / outcome / заметка, featured и visible, услуги, секции.

### Секции кейса (11 типов)
Текст, Цитата, Одно изображение, Галерея, Сетка изображений, Текст + изображение, Показатели,
Процесс / таймлайн, Видео (embed URL), Чеклист, До / После. Тип меняется выпадающим списком —
поля редактора подстраиваются автоматически.

### Разделы
Включение/выключение блоков сайта; списки «Брендинг» и «Арт-дирекшн» (с превью картинок и
привязкой к кейсу).

### Социовизуальное / Черновики / Креативные
Карточки с превью изображений, порядком и удалением.

### About / Contacts
Позиционирование, портрет (превью), компетенции, направления, опыт, email, telegram, услуги.

### Система (UI)
Название сайта, System ID, статус, эффекты noise / scanlines.

## Порядок и видимость

- Для массивов — кнопки `↑` / `↓` (порядок) и «Удалить».
- Снимите чекбокс «Показывать кейс», чтобы скрыть кейс с сайта, не удаляя его из данных.

## Видео

Для секции «Видео» вставляйте **embed URL** (например, YouTube/Vimeo embed).

## Лучший рабочий процесс

1. Редактируйте всё в `admin.html`, поглядывая в «Живое превью».
2. Проверьте результат.
3. Нажмите «Опубликовать на сайт» (или «Экспорт JSON» → замените `content/content.json`).
4. «Сбросить правки» вернёт исходный `content.json` (очистит локальные изменения).
