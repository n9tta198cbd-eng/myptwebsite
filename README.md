# N9TTA — сайт-портфолио

Готовый статический одностраничный сайт-портфолио с локальной редакторской панелью и data-driven контентной системой.

## Структура проекта

- `index.html` — сам сайт.
- `admin.html` — локальная редакторская панель.
- `assets/style.css` — вся дизайн-система, адаптивность и эффекты.
- `assets/main.js` — рендер сайта из `content/content.json` или локального override.
- `assets/admin.js` — визуальный редактор, экспорт / импорт JSON, локальное сохранение.
- `content/content.json` — весь контент сайта.
- `images/placeholders/*.svg` — временные заглушки, которые нужно заменить на финальные материалы.

## Быстрый запуск

### Вариант 1: просто открыть локально

1. Откройте папку проекта.
2. Запустите любой локальный сервер, чтобы `fetch()` корректно прочитал JSON.
3. Откройте `index.html`.

### Вариант 2: через VS Code Live Server

1. Откройте папку проекта в VS Code.
2. Установите расширение Live Server.
3. Нажмите `Go Live`.
4. Откройте `index.html` и `admin.html`.

### Вариант 3: через Python

```bash
cd portfolio-terminal
python -m http.server 8000
```

Затем откройте `http://localhost:8000`.

## Что уже реализовано

- Одностраничная архитектура с длинным вертикальным scroll-flow.
- Крупный hero-блок в эстетике архивного вычислительного терминала.
- Фиксированная навигационная колонка с якорями.
- 4 featured кейса.
- Инлайн-просмотр кейсов через drawer без перехода на отдельные страницы.
- Галереи, split-блоки, текстовые секции, цитаты, checklist, metricRow, processTimeline, before/after и video-секции.
- Адаптивность до мобильной ширины.
- Состояния focus, клавиатурная навигация, skip-link, lazy loading, reduced motion.
- Локальная редакторская панель без backend.
- Export / import JSON.
- LocalStorage preview override.
- Fallback-заглушки для отсутствующих изображений.

## Как заменить изображения

1. Положите свои файлы в папку `images/`.
2. В `admin.html` откройте нужный кейс.
3. Вставьте путь к новому файлу, например:
   - `images/cases/cyberclub/hero.jpg`
   - `images/cases/raindays/look-01.webp`
4. Экспортируйте новый `content.json`.
5. Подмените файл в `content/content.json`.

## Что лучше заменить в первую очередь

- Все файлы из `images/placeholders/`.
- Портрет в блоке `Обо мне`.
- Hero-изображения кейсов.
- Галереи внутри кейсов.
- Draft-описание кейса Gazgolder после уточнения фактов.

## Важное ограничение

Редактор не пишет файлы прямо на диск без backend — это нормально для статического сайта. Он:

- сохраняет локальные правки в браузере,
- даёт мгновенный предпросмотр на сайте,
- экспортирует готовый `content.json`.

Именно этот JSON затем нужно заменить в проекте или на хостинге.
